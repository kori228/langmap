# LangMap データレビュー #56 — 廣東話 (Cantonese / 粵語) 単一セル `yue` の書面語混入レビュー

## レビュアー自己紹介 (ペルソナ)

何家俊 (Ho4 Gaa1 Zeon3, handle: **@gaazeon_hkcantai**)。香港島西營盤 (Sai1 Jing4 Pun4) 生まれの典型的廣東話母語話者、家庭言語は 100% 廣東話 (廣府片・港式)、就学後に書面語 (modern Standard Written Chinese, MSWC) と英語をバイリンガル習得した HKDSE 中文科 (廣東話授業) 履修世代。香港中文大学中國語言及文學系出身、LSHK (香港語言學學會) 粵拼推廣員資格保持。常用参考: **粵語審音配詞字庫** (香港中文大学公開、https://humanum.arts.cuhk.edu.hk/Lexis/lexi-can/、以下「審音字庫」)、Matthews & Yip *Cantonese: A Comprehensive Grammar* 2nd ed. (Routledge 2011)、廖榮容 *廣州話正字典* (廣東人民出版社 1993)、麥耘・譚步雲 *實用廣州話分類詞典* (商務印書館 2011)、香港增補字符集 HKSCS-2016 文献。今回のレビュー軸は **「ユーザーが直接申し立てた『現状の `yue` データが実際に話される廣東話と乖離している』」** という決定的フィードバックの検証 — すなわち:

1. **書面語混入 (Mandarinism / 書面語直読症)** — 「是 / 不 / 沒 / 的 / 了 / 在 / 給 / 看 / 喝 / 吃 / 說 / 知道 / 漂亮 / 東西 / 來 / 怎麼 / 什麼 / 哪裡 / 多少 / 也 / 但是」等、書面語 (Standard Written Chinese, 北京語語法ベース) の語彙・助詞が「字音だけ粵讀 (Cantonese reading)」で持ち込まれていないか。粵字 (嘅・咗・喺・冇・唔・畀・睇・飲・食・佢) で書かれるべき箇所に書面語形が居座っていないか
2. **疑問助詞の北京語化** — 「嗎 (maa1)」は香港粵語で完全に書面語であり、口語の Yes/No 疑問は文末の「呀/啊/咩/嘛」または V-唔-V 構造、または「係咪」で表現される
3. **副詞接尾辞「地」** — 書面語の「形容詞 + 地 + 動詞」は粵語非対応、粵語は「形容詞 + 噉 (gam2)」または形容詞単独
4. **「我的」「我們」など完全書面語代名詞** — 廣東話は「我嘅」「我哋」が必須。「我的」「我們」は字音粵讀しても廣東話話者が発しないパターン
5. **量詞の北京語化** — 廣東話固有量詞 (碟・條・部・隻・份) が書面語量詞 (道・座・本) に置換されていないか
6. **アスペクト**: 完了「咗 (zo2)」が必須、書面語の「了 (liu5)」は誤り。進行「緊 (gan2)」が必須、書面語の「在 / 正在」は誤り

確信度マーク: 【確実】= 母語直感と審音字庫 / Matthews & Yip の記述が一致し、現行表記が日常廣東話として明確に不自然、【蓋然】= 文体差・register 差で迷い余地あり、【要検討】= 方針として closed-review で議論すべき。

レビュー #54 (台語 `nan` の北京語干渉) で陳怡君氏が指摘した defect class が、本 `yue` セルにも **同型かつ同強度で出現している**。とくにユーザーが「他の中国語系方言にも同類問題があるのでは」と明示的に疑念を表明している点を踏まえ、本レビューは指摘 #54 と同等のアグレッシブネスで臨む。

## 検証範囲

- **対象**: `data.js` 中の `yue` (廣東話/粵語) セル 全 100 文 (全件抽出済)
- **対照**: 審音字庫の見出し語形、香港 TVB 翡翠台・ViuTV 港式廣東話のラジオ・ドラマ字幕、香港教育大学《粵語拼音字表》、Matthews & Yip 2011 の正則語形
- **方法**: 各セルを (a) 書面語形 (是/不/沒/的/了/在/給/看/喝/吃/說/知道/漂亮/東西/來/怎麼/什麼/哪裡/多少/也/但是 + 「嗎」助詞 + 「地」副詞接尾) との照合、(b) 量詞・代名詞・アスペクトの粵語固有形照合、(c) 港式口語自然度 (自分の母語直感 + Hong Kong Cantonese Adult Language Corpus, HKCAC) の三軸でスキャン
- **read-only on data.js** — 提案は全て `{id, lang:"yue", op, ...}` 構造化 edit として記述、`.wf_langmap_apply.mjs` 仕様 (`op:"settext"` で `find`/`newText` + `allowTextChange:true` の語彙置換が大半) に従う

### 1. #92 `[A:我們]` — 完全書面語代名詞、廣東話は **我哋 (ngo5 dei6)** 【確実】

**Current:** `#92 [A:我們] [D:農曆新年] [C:食] [B:餃子]`

**Problem:** 「我們」(ngo5 mun4) は完全に書面語・普通話の 1 人称複数。廣東話の 1 人称複数は **我哋 (ngo5 dei6)** が唯一の口語形で、複数マーカーは **哋** (dei6、書面語の「們」とは別字)。Matthews & Yip 2011 §4.1.1 は「Cantonese 1pl pronoun is invariably 我哋, the form 我們 belongs to written register」と明記。香港の小学生でも「我哋食餃子」と言う、「我們食餃子」は中文教科書の朗読でしか聞かれない。本データ内 #54 `[A:我哋]` (我們必須保護環境) では正しく **我哋** を採用しているのに、#92 だけ「我們」になっている **内部不整合の典型例**。さらに #92 全体は「我哋過年食餃子」が自然 (農曆新年は書面語、口語は **過年** gwo3 nin4)。最小修正は「我們→我哋」。

**Proposed edit:**
```json
{"id":92, "lang":"yue", "op":"settext", "find":"我們", "newText":"我哋", "allowTextChange":true}
```

### 2. #86 `[B:我的智能手機]` — 「我的」は書面語、廣東話は **我嘅 (ngo5 ge3)** 【確実】

**Current:** `#86 [D:我] [A:每日] [C:用] [B:我的智能手機]`

**Problem:** 廣東話の所有・属性マーカーは **嘅 (ge3)** が唯一の口語形、書面語の「的」(dik1) とは完全に別字 (審音字庫: 嘅 = 粵語固有字、HKSCS 収録)。**「我的」は字音粵讀しても廣東話話者が一度も発しないパターン** (Matthews & Yip 2011 §6.3.1)。本データ内では他のほぼすべての所有・修飾セルで「嘅」を採用済 — #2 `[A:我嘅]`、#4 `[A:最近嘅]`、#12 `[B:公園嘅]`、#15 #38 #43 #49 #51 #65 #71 #76 #80 #91 で **嘅** が正しく使われている。#86 だけ「我的」と書面語に滑っている **完全な内部不整合**。さらに「智能手機」も普通話寄り (港式口語は単に **手機** sau2 gei1 または **電話** din6 waa2)、ただし最小修正は「我的→我嘅」。

**Proposed edit:**
```json
{"id":86, "lang":"yue", "op":"settext", "find":"我的智能手機", "newText":"我嘅手機", "allowTextChange":true}
```
(保守的最小修正なら `find:"我的", newText:"我嘅"`)

### 5. #47 `[C:清楚地]` / #57 `[C:流利噉]` — 副詞接尾辞「地」は書面語、廣東話は **噉 (gam2)** 【確実】

**Current:**
- #47 `[A:老師] [C:清楚地] [D:解釋了] [B:道題]`
- #57 `[A:佢] [C:流利噉] [D:講] [B:三種語言]`

**Problem:** 書面語の副詞化接尾辞「地」(de/dik6) は廣東話に **存在しない** (Matthews & Yip 2011 §10.3.4)。廣東話は **噉 (gam2)** = 「〜のように」を形容詞に後接させて副詞化する: 「清楚噉解釋」「流利噉講」。本データ内 #57 では **流利噉** で正しく「噉」を採用しているのに、#47 だけ「清楚地」と書面語接尾辞が使われている **内部不整合**。「老師清楚噉解釋咗條題」(lou5 si1 cing1 co2 gam2 gaai2 sik1 zo2 tiu4 tai4) が正則。「地」を「噉」に置換するか、または廣東話副詞は形容詞単独でも機能するため (例:「快啲講」=「速く言って」) 削除して「清楚解釋」でも可。

**Proposed edit:**
```json
{"id":47, "lang":"yue", "op":"settext", "find":"清楚地", "newText":"清楚噉", "allowTextChange":true}
```

### 6. #28 `[G:嗎]` / #68 `[D:嗎]` — 疑問助詞「嗎」は完全書面語、廣東話は **呀/啊/咩** または V-唔-V 【確実】

**Current:**
- #28 `[B:你] [A:可以] [C:介紹] [D:間] [F:好嘅] [E:餐廳] [G:嗎]`
- #68 `[A:我] [D:可以] [C:坐] [B:喺度] [D:嗎]`

**Problem:** 書面語の Yes/No 疑問助詞「嗎」(maa1) は廣東話で **絶対に使われない** (Matthews & Yip 2011 §17.1.2)。廣東話の Yes/No 疑問形は:
1. **V-唔-V**: 「可唔可以」「得唔得」(本データ #10 `[D:識] [E:唔識]` で正しく採用済)
2. **文末助詞 呀 (aa3) / 啊 (aa1) / 咩 (me1)**: 「可以介紹嗎?」→「可以介紹呀?」、「可以坐喺度嗎?」→「坐喺度得唔得呀?」
3. **係咪 (hai6 mai6) 構造**: 「係咪可以...?」

本データ #10 `[A:你][D:識][E:唔識][C:講][B:英文][E:無]` (* `[E:無]` は無くてよい、後述) で V-唔-V を正しく採用、#40 `[E:知唔知]` でも V-唔-V を採用しているのに、#28 と #68 だけ「嗎」を残している **内部不整合**。香港の店員に「呢度可以坐嗎?」と言うと「あ、外国人だ」と思われる typical 書面語間違い。

**Proposed edits:**
```json
{"id":28, "lang":"yue", "op":"settext", "find":"嗎", "newText":"呀", "allowTextChange":true}
{"id":68, "lang":"yue", "op":"settext", "find":"嗎", "newText":"呀", "allowTextChange":true}
```
(より積極的には #28「可以」→「可唔可以」、#68「可以坐」→「坐唔坐得」だが letter 構造変更を伴うため最小修正は文末助詞置換)

### 8. #49 `[D:告訴]` — 「告訴」は書面語、廣東話は **話 (waa6) ... 知** または **同 ... 講** 【確実】

**Current:** `#49 [A:請] [D:告訴] [B:我] [F:去] [G:機場] [E:嘅路]`

**Problem:** 「告訴」(gou3 sou3) は廣東話で「(法廷で) 訴える」の語義のみで使われ、「教える・伝える」の意では **絶対に使わない** (審音字庫: 告訴 = legal term, written register)。日常廣東話の「(私に) 教える」は:
1. **「話 + 對象 + 知」**: 「話我知」(waa6 ngo5 zi1) = 「私に教える」
2. **「同 + 對象 + 講」**: 「同我講」(tung4 ngo5 gong2) = 「私に向かって言う」

「請告訴我去機場嘅路」は書面語の朗読、廣東話の自然形は **「請話我知去機場點行」(cing2 waa6 ngo5 zi1 heoi3 gei1 coeng4 dim2 haang4)** または **「請同我講去機場嘅路」**。ユーザーが申し立てた「乖離」の中核例で、台語レビュー #54 指摘 2 と完全に同型 (台語: 告訴 → 講; 粵語: 告訴 → 話...知 / 同...講)。

**Proposed edit:**
```json
{"id":49, "lang":"yue", "op":"settext", "find":"告訴", "newText":"話", "allowTextChange":true}
```
(最小修正は「告訴 → 話」、より自然には segment 構造を変えて `[D:話][B:我][?:知]` だが letter 増加するため最小修正に留める)

### 32. 粵拼 (Jyutping) 注記の全面不在 — メタ指摘、grand convention 議題 【要検討】

**Current:** 全 100 文の `yue` セルが **漢字のみ**、粵拼 (Jyutping) または別系統羅馬字 (Yale / Sidney Lau) の注記ゼロ。

**Problem:** 廣東話の表記体系は **漢字 + 粵拼 (Jyutping, LSHK 1993 標準)** の併記が学習者向け資料の現代標準 (香港教育大学《粵語拼音字表》、香港中文大学《粵語審音配詞字庫》、Matthews & Yip 2011 全章で Jyutping 併記)。漢字のみでは:
- **声調 (6 tone) が判別不能**: 同字「重」(zung6 = heavy / cung4 = repeat) の判別不能
- **入声韻尾 (-p/-t/-k) の判別不能**: 標準中文輸入では区別されない
- **異読 (literary vs colloquial reading) の指定不能**: 「人」(jan4 / ngan4)、「日」(jat6 / jat2) 等
- **HKSCS 拡張字 (嘅・咗・喺・冚・哋 等) の入力支援が皆無**: ユーザーがコピペできない

理想は `yue` セルの各 segment に `[A:我|ngo5]` のように **漢字|Jyutping** ペアで表記。これは LangMap data schema 拡張を要する **grand convention** で本レビュー単独では決定不能。closed-review にて:
- **Q1**: `yue` セルに Jyutping フィールドを追加するか? Jyutping (LSHK) / Yale / Sidney Lau のいずれを採用するか? (LSHK Jyutping 推奨、最も中立かつ Unicode 親和)
- **Q2**: 台語 (#54 指摘 29、POJ/Tâi-lô) と同様の声調言語家族との表記方針対称
- **Q3**: 他の声調言語 (vi / th / lo / wuu / hak / cmn) との表記方針対称

を議論されたい。本レビューでは方針提案のみ、データ修正なし。
