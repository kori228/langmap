# LangMap データレビュー #54 — 台語 (Tâi-gí / Taiwanese Hokkien) 単一セル `nan` の文白二読・北京語干渉レビュー

## レビュアー自己紹介 (ペルソナ)

陳怡君 (Tân Î-kun, handle: **@taigi_chiakun**)。台南生まれの台語母語話者、家庭言語は 100% Tâi-gí、就学後に華語 (Mandarin) を習得した典型的な「漳泉混合・南部腔」話者。台湾師範大学台湾文化及語言文学研究所修士、教育部認証台羅 (Tâi-lô) 教師資格保持。常用参考: **教育部臺灣閩南語常用詞辭典** (https://twblg.dict.edu.tw/、以下「教典」)、Klöter *Written Taiwanese* (Harrassowitz 2005)、鄭良偉 *演變中的台灣社會語文* (自立晚報 1990)、楊允言・張學謙・呂美親 編 *台語文運動訪談暨史料彙編* (國史館 2008)、洪惟仁 *台灣方言之旅* (前衛 1992)、村上嘉英 *東方台湾語辞典* (東方書店 2007)。今回のレビュー軸は **「ユーザーが直接申し立てた『現状の `nan` データが実際に話される台語と乖離している』」** という決定的フィードバックの検証 — すなわち:

1. **北京語直訳由来の語彙混入 (Mandarinism)** — 「喜歡 / 告訴 / 正在 / 漫遊 / 多 / 道 (量詞) / 一個 / 部 (量詞) / 給」等、現代台湾華語の語彙が「字音だけ閩南読み」で持ち込まれていないか
2. **文読 (literary 文讀) ↔ 白読 (vernacular 白讀) の選択** — 「日語/日本話、父母/爸母、廟宇/廟、故事/古、經歷/經驗、改善/改進、舉辦/舉行」等で、辞書形 (文讀) ばかりが選ばれて日常口語 (白讀) が回避されていないか
3. **量詞 (measure word) の北京語化** — 台語の慣用量詞は「齣 (tshut, 電影/戲)」「條 (tiâu, 橋/路/魚)」「个 (ê, 一般)」だが、北京語の「部・道・座・個 (簡体含意)」が混入していないか
4. **親族呼称の人称代名詞** — 台語は所有「私の母」を **阮母 / 阮老母 / 阮 + 親族** で表す (gún + 親族、Norman 1988 *Chinese* §10.3.2)。「我媽媽 / 我小妹 / 我貓」のような **「我 + 親族/所有」は北京語直訳パターン** で、本来の台語に存在しない
5. **アスペクト・進行形** — 進行は **咧 (leh)** が必須、北京語の「正在」を流用してはならない
6. **羅馬字 (POJ / Tâi-lô) の併記不在** — データセット全体で羅馬字注記がゼロ。これは台語表記の半身を欠く深刻な情報損失だが、これは grand convention 案件としてオーナー裁定議題に昇格させる

確信度マーク: 【確実】= 母語直感と教典記述が一致し、現行表記が日常台語として明確に不自然、【蓋然】= 文体差・地域差で迷い余地あり、【要検討】= 方針として closed-review で議論すべき。

## 検証範囲

- **対象**: `data.js` 中の `nan` (Taiwanese Hokkien) セル 全 100 文 (`node -e ".../return SENTENCES;"` で全件抽出済)

### 1. #19 `[D:喜歡]` — 北京語「喜歡」が台語に混入、本来は **愛 (ài) / 佮意 (kah-ì)** 【確実】

**Current:** `#19 [A:我] [D:喜歡] [C:聽] [B:音樂]`

### 7. #8 `[A:我媽媽]` / #15 `[A:我小妹]` / #50 `[E:我的] [A:狗]` / #73 `[E:我] [A:貓]` — 「我 + 親族・所有」は北京語直訳、台語は **阮 (gún)** 【確実】

**Current:**
- #8 `[A:我媽媽] [D:每工] [C:煮] [B:暗頓]`
- #15 `[A:我小妹] [D:咧] [B:佇法國] [C:留學]`

- (参考: #25 `[A:我老爸]` も同型)

**Problem:** 台語の所有 1 人称複数 **阮 (gún)** は「私たちの」と同時に「私の (親族・身内・所有物)」を表す **inclusive/exclusive 兼用**の特殊代名詞 (Norman 1988 *Chinese* §10.3.2; 鄭良偉 1990 §4.2)。家族・住居・ペットなど「身内」概念には **阮** が必須で、「我媽媽」(guá ma-ma) のような「我 + 親族」は **北京語「我媽媽」の文字面コピー** にすぎない。台南・高雄の母語話者は:
- 「私の母」→ **阮母 / 阮母仔 / 阮老母** (gún bú / gún bú-á / gún lāu-bú)
- 「私の妹」→ **阮小妹** (gún sió-muē)
- 「私の犬」→ **阮狗 / 阮的狗** (gún káu)
- 「私の猫」→ **阮貓 / 阮的貓** (gún niau)

### 10. #85 `[C:多啉]` — 「多」+「啉」のハイブリッド、台語は **啉較濟 (lim khah-tsē)** 【確実】

### 29. 羅馬字 (POJ / Tâi-lô) 注記の全面不在 — メタ指摘、grand convention 議題 【要検討】

**Current:** 全 100 文の `nan` セルが **漢字のみ**、羅馬字注記ゼロ。

**Problem:** 台語の表記体系は **漢字 + 羅馬字 (POJ または Tâi-lô) の併記** が教育部・台文界の標準 (Klöter 2005 §7)。学習者向けデータセットとして:
- 漢字のみでは **声調 (8 tone) が判別不能**:同字「重」(tāng = heavy / tîng = repeat) の判別不能
- 漢字のみでは **文白二読の指定不能**:「人」(jîn 文 / lâng 白)、「日」(ji̍t / li̍t / ja̍t 方言差) 等
- 漢字のみでは **方言差 (泉/漳/廈) の指定不能**

理想は `nan` セルの各 segment に `[A:我|guá]` のように **漢字|羅馬字** ペアで表記。これは LangMap データ schema 拡張を要する **grand convention** で本レビュー単独では決定不能。closed-review にて:
- **Q1**: `nan` セルに羅馬字フィールドを追加するか? Tâi-lô / POJ いずれの schema を採用するか?
- **Q2**: 既存の中国語系セル (zh-cmn 等) にも拼音注記を併記するかとの整合性
- **Q3**: 他の声調言語 (vi / th / lo / yue / wuu / hak) との表記方針対称

を議論されたい。本レビューでは方針提案のみ、データ修正なし。
